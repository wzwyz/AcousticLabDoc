节点介绍

如上所示的块称为节点。节点左侧与右侧有相应颜色的圆圈，称为节点的端口，颜色指代节点接收或发出的数据类型，左边的端口是输入端口，右边的端口是输出端口。 除非特别注明某些输入端口是同步端口，否则当任意与输入端口连接的上一节点发生更新，本节点也会立即跟新。

Wave2Graph节点

输入 ：一路Sound
实现功能 ：在3D绘图区延Z轴和X轴依次绘制数据包中的点。

Result节点

输入 ：一路Decimal
实现功能 ：显示输入的Decimal的数值。

VectorDisplay节点

输入 ：一路Vector3D
实现功能 ：显示输入的Vector3D的数值。

Oscillograph节点

输入 ：一路Sound
实现功能 ：在绘图区新建一个2D绘图窗口，其中按照采样率依次绘制Sound包内的音频序列。
操作界面 ：按钮上显示的文字即为2D绘图窗口的名字，点击按钮，可以显示和隐藏对应的绘图窗口。

Multi-Channel Oscilloscope节点

输入 ：多路Sound ，可以在操作界面相应添加、减少
实现功能 ：在绘图区新建一个2D绘图窗口，其中按照采样率用不同颜色依次绘制各个Sound包内的音频序列。
操作界面 ：其上显示的文字即为2D绘图窗口的名字；点击按钮+ ，可以增加输入的端口数量；点击按钮- ，可以将最近添加的一个端口删除；点击设置按钮，可以显示和隐藏对应的绘图窗口。

如果输入的Sound包的采样率为0或为负，则Oscillograph节点和Multi-Channel Oscilloscope节点将不能正确地绘制图像！

User Custom节点

输入 ：用户在配置窗口定义
输出 ：用户在配置窗口定义
实现功能 ：打开一个新的ExprTK用户配置窗口，并加载和执行用户指定的ExprTK代码段
操作界面 ：按钮上显示的文字即为ExprTK用户配置窗口的名字，点击按钮，可以显示和隐藏对应的配置窗口，配置窗口的介绍详蓝图编辑器说明。文字则表明模块当前的状态，可能的状态有未编译、正常、超时和运行时错误。当本次执行本模块程序超过1秒时，状态变为超时。若出现超时和运行时错误和运行时错误提示，请用户仔细检查代码并重新编译。

Division节点

输入 ：一路Decimal表示除数，一路Decimal表示被除数，具体请按端口标题相应连接。 输出 ：一路Decimal 实现功能 ：输出被除数和除数相除所得的数值。

Additon节点

输入 ：两路Decimal
输出 ：一路Decimal
实现功能 ：输出两数相加所得的数值。

Subtraction节点

输入 ：一路Decimal表示减数，一路Decimal表示被减数，具体请按端口标题相应连接。 输出 ：一路Decimal
实现功能 ：输出两数相减所得的数值。

Magnitude节点

输入 ：一路ComplexSound
输出 ：一路Sound
实现功能 ：将ComplexSound包中的虚数序列依次求模并填入Sound包中，两包的采样率保持一致。

Buffer节点

输入 ：一路Decimal
输出 ：一路Sound
实现功能 ：每当输入的Decimal更新的时候，就将其填入到Sound包的头部，输出Sound包的采样率默认为42kHz。
应用场景 ：见示例多普勒效应的可视化

在上述模块中出现了从双精度向16位整数的转换，会发生数据的丢失与可能的溢出，如果发生溢出，可以使用User Custom节点编写模块将Sound包中的数值均除以一个适当的常数。

Multiplication节点

输入 ：两路Decimal
输出 ：一路Decimal
实现功能 ：输出两数相乘所得的数值。

FFT节点

输入 ：一路Sound
输出 ：一路ComplexSound
实现功能 ：输出对Sound包中的声音序列做傅里叶变换后的结果。

IFFT节点

输入 ：一路ComplexSound
输出 ：一路Sound
实现功能 ：输出对ComplexSound包中的复数序列做逆傅里叶变换后的结果。

OctaTDOALocator节点

输入 ：一路Decimal表示金字塔型阵列的阵元半径，五路Decimal依次表示其余五个阵元M1~M5与中心阵元M0收到声音的延迟。
输出 ：一路Vector3D
实现功能 ：输出对传入的延迟数据使用TDOA算法进行声源定位的结果。

ComplexCalculate节点

输入 ：两路ComplexSound
输出 ：一路ComplexSound
实现功能 ：输出对传入的两路ComplexSound执行互相关算法的结果。

Index节点

输入 ：一路Sound
输出 ：一路Deciaml
实现功能 ：在传入的Sound包内涵的声音序列中找到最大值，并输出最大值所处的位置。

VectorSource节点

输出 ：一路Vector3D
实现功能 ：将用户填入的3个分量打包为Vector3D并输出。
操作界面 ：三个数字输入框，分别表示x、y、z轴的分量。

NumberSource节点

输出 ：一路Decimal
实现功能 ：将用户的十进制数打包为Decimal并输出。
操作界面 ：一个数字输入框。

SoundSource节点

输出 ：六路Sound
实现功能：

  1. 读取用户指定的mp3文件左声道的第一帧。

  2. 假设此音频是金字塔型阵列中M0阵元收到的麦克风信号，依次计算剩余五个麦克风对此麦克风的延迟并制作出对应的音频。

  3. 计算每段音频的均方根功率，并以20的信噪比添加高斯噪声。

  4. 若处在播放状态，则每隔4096个采样周期从上述六段音频中读取4096个采样点打包成Sound包输出。

  5. 若音频剩余采样值小于4096，则停止播放。


    操作界面 ：操作界面由播放按键、播放进度条、打开文件按键与文件信息显示框组成

  • 点击播放按键则开始上述播放状态，进度条显示播放的进度

  • 点击打开文件按钮即可选择mp3文件，若选择的文件格式正确，则文件信息显示框中将能正确显示音频文件的采样率和第一帧的数据长度。 否则，将显示错误。

RemoteSoundSource节点

输出 ：多路Sound ，由用户自定义。
实现功能 ：每当收到新的数据时，解包多功能采集终端发送的数据，并按照输入的通道打包为Sound。
操作界面 ：由多个通道指定控件和设备选择框组成。通道指定控件依次由端口输入框、添加按钮和移除按钮构成，可以在通道输入框内填入0~15来决定其对应的端口所输出数据的对应通道。点击添加则可在列表的末尾添加一的新的端口指定控件，点击移除则删除对应控件与端口。请对照设备检测窗口中列出的ip将其填入设备选择框中，这样当设备检测窗口中对应设备的开始采集按钮按下的时候，本节点开始输出数据。
